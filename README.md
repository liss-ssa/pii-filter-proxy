# Russian PII Proxy MVP

Локальный preprocessing-прокси для обезличивания персональных данных перед передачей текста в LLM. Сервис работает полностью локально и поддерживает обработку plain text, Markdown, HTML, TXT, DOCX и PDF с текстовым слоем. OCR и изображения в MVP не поддерживаются.

## Архитектура

Конвейер состоит из четырёх основных этапов.

### 1. Детерминированное обнаружение

Regex и валидаторы обнаруживают структурированные персональные данные:

- email;
- телефоны;
- паспорт РФ;
- СНИЛС;
- ИНН физического лица;
- банковские карты;
- банковские счета;
- IP-адреса.

Для ИНН, СНИЛС и банковских карт используются проверки контрольных сумм, что снижает количество ложных срабатываний.

### 2. GLiNER2-PII

Основной NER-моделью является **GLiNER2-PII** (`fastino/gliner2-privacy-filter-PII-multi`), работающая полностью локально.

Модель используется для обнаружения:

- ФИО;
- дат;
- паспортных данных;
- телефонов;
- email;
- банковских реквизитов;
- IP-адресов;
- других персональных сущностей.

После первого скачивания модели весь инференс выполняется локально.

### 3. Контекстный классификатор

Контекстный классификатор на основе **ruBERT-tiny2** анализирует неоднозначные случаи и классифицирует найденные сущности как:

- `PII`;
- `NOT_PII`;
- `UNCERTAIN`.

Например, классификатор помогает отличать персональные идентификаторы от похожих числовых последовательностей.

### 4. Policy Engine

Все найденные сущности проходят через Policy Engine.

Он:

- объединяет результаты Regex, GLiNER2 и классификатора;
- исключает известные ложные срабатывания;
- применяет fail-safe политику для неоднозначных случаев;
- генерирует стабильные placeholders внутри документа.

Все даты используют единый тип:

```
DATE
```

Повторяющиеся значения получают одинаковый placeholder:

```
Иванов Иван Иванович → [PERSON_1]
Иванов Иван Иванович → [PERSON_1]
```

Организации по умолчанию не маскируются.

## Подготовка моделей

При первом запуске требуется загрузить модели с Hugging Face.

```bash
python scripts/download_models.py
```

После скачивания дальнейшая работа полностью локальная.

Для обучения контекстного классификатора:

```bash
python scripts/generate_context_dataset.py --rows 3000
python scripts/train_context_classifier.py --epochs 3 --batch-size 32
```

Для Windows с NVIDIA RTX:

```powershell
python -c "import torch; print(torch.cuda.is_available())"
```

При отсутствии CUDA автоматически используется CPU.

---

## Конфигурация

```env
NER_BACKEND=gliner2

GLINER2_MODEL_NAME=fastino/gliner2-privacy-filter-PII-multi
GLINER2_THRESHOLD=0.55

CONTEXT_CLASSIFIER_ENABLED=true
CONTEXT_MODEL_DIR=models/context-rubert-tiny2

UNCERTAIN_POLICY=mask

MASK_ORGANIZATION_PII=false

INFERENCE_DEVICE=auto
```

`INFERENCE_DEVICE=auto` использует CUDA при наличии, иначе CPU.

## Ограничения MVP

Текущая версия проекта имеет несколько известных ограничений:

- модель GLiNER2 обучалась преимущественно на европейских языках и требует дополнительной оценки на русскоязычных данных;
- синтетические метрики подтверждают корректность работы pipeline, но не являются оценкой качества на реальных документах;
- качество должно подтверждаться независимым размеченным набором русскоязычных документов.
